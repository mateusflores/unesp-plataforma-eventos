Cronograma do Projeto

Acompanhamento das fases de desenvolvimento da Plataforma de Ingressos.

Fase 1

Planejamento e ambiente

06/04 – 19/04

Todos juntos

  • Levantar requisitos e definir escopo mínimo
  • Criar repositório no GitHub com branches back/front
  • Montar o Trello com as tarefas
  • Esboçar o diagrama UML das entidades principais (Evento, Usuário, Ingresso)

Back-end (2-3 pessoas)

  • Configurar projeto Spring Boot
  • Definir as entidades JPA e o banco H2 ou MySQL

Front-end (2-3 pessoas)

  • Criar projeto React com Vite
  • Instalar Bootstrap e definir paleta visual
Fase 2

Funcionalidades principais

20/04 – 17/05

Back-end

  • API REST: cadastro e login de usuário (JWT ou sessão)
  • API REST: CRUD de eventos (criar, listar, editar, excluir)
  • API REST: compra de ingresso e listagem por usuário
  • API REST: painel do organizador (ver participantes)

Front-end

  • Tela de login e cadastro
  • Listagem de eventos em formato lista e calendário simples
  • Tela de detalhe do evento com botão de compra
  • Formulário de criação de evento
  • Painel do organizador (lista de participantes)
Fase 3

Integração e ajustes

18/05 – 07/06

Todos juntos

  • Integrar front e back (ajustar CORS, contratos da API)
  • Testar os fluxos principais de ponta a ponta
  • Corrigir bugs identificados na integração

Extras se der tempo

  • Filtro de eventos por categoria
  • Tela "meus ingressos" para o usuário
  • Validações e mensagens de erro amigáveis
Fase 4

Entrega e apresentação

08/06 – 28/06

Todos

  • Polir a interface e corrigir inconsistências visuais
  • Preparar slides e demo para apresentação
  • Escrever README com instruções de uso
  • Deploy simples (Railway, Vercel, ou Netlify)
